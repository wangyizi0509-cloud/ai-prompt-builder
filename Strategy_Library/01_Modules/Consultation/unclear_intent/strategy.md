# 意图不明策略 (Unclear Intent Strategy)

## 1. 模块定位
*   **角色**：引导者
*   **核心目标**：处理无法判断意图的Query，引导用户说明来意
*   **输入**：被分类为 `UNCLEAR` 的用户 Query
*   **输出**：友好的追问/引导

## 2. 触发场景

| 场景 | 典型示例 |
|:---|:---|
| **信息过少** | "你好"、"在吗"、"嗯"、"..." |
| **陈述无目的** | "我今天见了她一面" |
| **表达模糊** | "就那样吧"、"唉" |

## 3. 回复策略

### 3.1 核心原则

- **友好开放**：不要让用户觉得被审问
- **降低门槛**：给出选项，让用户更容易回答
- **保持人设**：用军师的语气，轻松自然

### 3.2 回复结构

**方式一：开放式追问**
- 用于用户刚打招呼的情况
- 给用户完全的自由度

**方式二：选项式引导**
- 用于用户说了点什么但不够清楚的情况
- 给出几个可能的方向，降低回答成本

---

## 4. 二次追问策略

如果用户第一次追问后仍然回复模糊：

**处理方式**：给出更具体的选项

---

## 5. 通用原则

| 原则 | 说明 |
|:---|:---|
| **热情但不过度** | 欢迎用户，但不要太客服化 |
| **不要催促** | 给用户时间组织语言 |
| **引导不审问** | 问法要自然，不要像填表 |
| **保持耐心** | 即使用户多轮不说清，也不要表现出不耐烦 |

## 6. 边界情况

| 情况 | 处理方式 |
|:---|:---|
| 多轮追问后仍不说明 | 保持开放，可以主动抛一个话题 |
| 用户明显不想聊 | 尊重用户，给个台阶 |
| 追问后转入明确意图 | 正常进入对应策略处理 |
