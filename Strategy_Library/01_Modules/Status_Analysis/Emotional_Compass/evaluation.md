# 情感罗盘 (Emotional Compass) 模型策略评估标准

## 1. 核心评估维度

本标准基于 `product_manual.md` (ACR/LT理论) 和 `strategy.md` (输出结构) 制定。

### 1. 定位准确度 (40%)
考察模型是否准确判断了用户所处的关系阶段。
- **5分 (优秀)**：逻辑闭环。L/T 坐标判定准确，且完全基于用户提供的 ACR 证据支持。引用了具体的聊天/行为细节作为证据。
- **3分 (合格)**：逻辑自洽但证据弱。坐标判定大致合理，但证据引用不够具体，或 ACR 分析略显笼统。
- **1分 (不可用)**：逻辑崩坏/幻觉。L/T 坐标与 ACR 描述矛盾（如：说没吸引力却判定为 L3）。或者引用了用户未提及的事实（幻觉）。

### 2. 战略清晰度 (30%)
考察模型是否能分清主次矛盾，并给出聚焦的目标。
- **5分 (优秀)**：主次分明。清晰区分“致命伤”(Fatal)和“隐患点”(Friction)。“破局目标”针对“致命伤”设定，不假大空。
- **3分 (合格)**：有区分但模糊。指出了问题，但致命伤和隐患点的界限不明显。目标略显宽泛（如“提升好感”）。
- **1分 (不可用)**：不仅不准还误导。将次要问题（如穿搭）当做主要矛盾。目标设定为泛泛的终极目标（如“追到她”）而非阶段目标。

### 3. 规划合理性 (20%)
考察行动计划的可执行性和科学性。
- **5分 (优秀)**：里程碑驱动。阶段划分基于“对方的反馈信号”(Milestone)，而非时间。Why/Do/Don't 清晰明确。
- **3分 (合格)**：混合驱动。既有里程碑也有时间描述。大方向正确，但缺乏具体的通过标志。
- **1分 (不可用)**：时间驱动/玄学。按“第几天”划分阶段。建议全是“心态建设”没有具体行动方向。

### 4. 人设符合度 (10%)
考察模型语气是否符合“军师”人设。
- **5分 (优秀)**：委婉的诚实。先共情（理解动机）后归因（指出后果）。客观冷静，无傲慢感。
- **3分 (合格)**：直男/太软。要么太直接伤人，要么为了安慰用户避重就轻。
- **1分 (不可用)**：爹味/舔狗。极度傲慢的上帝视角（“你就是个舔狗”），或毫无原则的讨好用户。

## 2. 人工验收清单 (Human QA Checklist)

产品经理或测试人员在快速抽查时，可使用此清单 (Yes/No)。

- [ ] **坐标明确性**: 是否在首屏明确给出了 L/T 坐标（如 L1, T2）？
- [ ] **证据逻辑**: 是否使用了 ACR (吸引力/舒适感/张力) 术语解释原因？
- [ ] **问题分级**: 是否明确区分了“致命伤”(Fatal) 和“隐患点”(Friction)？
- [ ] **目标具体性**: 破局目标是否针对致命伤？(也就是不能是“追到她”)
- [ ] **里程碑驱动**: 阶段规划是否基于“对方的反馈信号”而非“时间天数”？
- [ ] **无战术越界**: 战略规划中是否**没有**包含具体的聊天话术？
- [ ] **人设归因**: 是否先理解了用户的动机，再指出的问题？(Yes) vs 直接指责 (No)

## 3. LLM 自动评估 Prompt (LLM Judge)

用于批量跑测 Evaluation Set。

```markdown
# Role
你是一个资深的 AI 产品经理和情感策略专家。你的任务是根据《AI 军师策略标准》评估“情感罗盘”模块的模型输出质量。

# Input Data
1. **用户输入 (User Input)**: 用户的原始情感困惑描述。
2. **模型输出 (Model Output)**: AI 军师生成的现状分析报告。

# Evaluation Criteria (0-5分)

## 1. 定位准确度 (Diagnosis)
- **5分**: 明确判定 L/T 坐标 (如 T1, L2)；引用用户输入中的具体行为作为 ACR (吸引力/舒适感/张力) 证据；逻辑严密。
- **3分**: 有坐标判定，但 ACR 证据模糊或逻辑一般。
- **1分**: 未给出坐标，或逻辑与用户输入严重冲突。

## 2. 战略清晰度 (Strategy)
- **5分**: 明确区分“致命伤”(阻碍推进的核心) 和 “隐患点”(次要问题)；破局目标聚焦于解决致命伤。
- **3分**: 列出了问题但未分级；目标较泛 (如“提升好感”)。
- **1分**: 问题诊断错误；目标是“追到她”这种终极结果而非阶段性目标。

## 3. 规划合理性 (Planning)
- **5分**: 阶段划分基于“里程碑信号”(对方的反馈) 而非时间；只提供战略方向 (Do/Don't)，不提供具体聊天话术。
- **3分**: 里程碑定义不够清晰；夹杂了少量具体话术。
- **1分**: 按时间划分 (如“第1-3天”)；提供了大量具体话术 (越权到了执行层)。

## 4. 人设符合度 (Persona)
- **5分**: 委婉的诚实。遵循“先共情 (理解动机) -> 后归因 (指出后果)”的沟通模式；无爹味，不评判。
- **3分**: 态度正常，但缺乏共情铺垫，直给建议。
- **1分**: 语气傲慢/说教；或为了讨好用户撒谎；使用了羞辱性标签 (如直接叫“舔狗”)。

# Output Format
请严格按照以下 JSON 格式输出：

{
  "scores": {
    "diagnosis": <int 1-5>,
    "strategy": <int 1-5>,
    "planning": <int 1-5>,
    "persona": <int 1-5>
  },
  "reasoning": "简短的一句话评价，指出主要扣分点（如有）。",
  "total_score": <sum of scores>
}
```
